# Roadmap покращення Pullora

Оновлено: 17 липня 2026 року.

Цей файл є джерелом правди для запланованих покращень Pullora після `v5.2.43`. План не залежить від історії чатів або пам'яті асистента.

## Основні правила

- Українська є мовою за замовчуванням.
- Усі нові тексти повинні мати український та англійський переклади.
- Не змінювати усталений темний дизайн без окремого рішення.
- Спочатку надійність і безпека, потім нові можливості.
- Не додавати залежності, якщо завдання вирішується засобами React, TypeScript, Rust або Tauri.
- Кожен користувацький реліз повинен пройти production-збірку, smoke-test і перевірку обох мов.

## Незмінний дизайн-контракт

Канонічний візуальний baseline Pullora — реліз `v5.10.1`, який відновлює перевірений дизайн `v5.9.0`. Усі наступні роботи з UI є внутрішньою уніфікацією, виправленням дефектів або оптимізацією, але не редизайном.

Обов'язково зберігати без візуальних змін:

- прозорі й розмиті підкладки, крізь які помітний загальний фон;
- чинну синю палітру, сині акценти, selection, focus і primary-дії;
- поточну композицію Library, ширину sidebar, hero card, панель основної дії, блоки версій і деталей;
- чинні відступи, висоти, радіуси, типографіку, іконки, щільність і візуальну ієрархію;
- поточний вигляд Settings, About, Install, контекстних і випадаючих меню;
- окремі темну та світлу теми як дві реалізації однієї айдентики, а не два різні продукти;
- незалежність загального фону, hero-фону й обкладинки застосунку.

Заборонено без окремого рішення користувача:

- змінювати палітру, стиль, композицію або пропорції в межах технічного рефакторингу;
- глобально перефарбовувати поверхні кольором обкладинки, hero-фону чи власного фону;
- замінювати наявну розмітку або CSS новою дизайн-системою одним великим кроком;
- вважати скорочення CSS достатньою причиною для зміни зовнішнього вигляду;
- позначати UI-пункт завершеним без порівняння з baseline-знімками.

Допустимі візуальні зміни: лише явно замовлене виправлення конкретного дефекту, доступність, читабельність або адаптація до підтримуваного розміру вікна. Така зміна повинна бути локальною й не змінювати решту екрана.

  - [x] вибір активного застосунку;
  - [x] фільтрація й сортування;
  - [x] масові оновлення.

Додатково:

- [x] Централізувати повторювані обчислення статусів.
- [x] Додати компонентні тести основних панелей.
- [x] Зафіксувати мінімальні й максимальні розміри карток.
- [x] Не вводити нову глобальну state-бібліотеку без доведеної потреби.

Архітектурне рішення: локальний стан компонентів і тематичні hooks покривають поточні сценарії; нова глобальна state-бібліотека не потрібна.

Готово, коли компоненти можна тестувати окремо, а `LibraryPage` відповідає переважно за компонування сторінки.

## v5.10.0 — єдина дизайн-система

Мета: усі теми, вікна, меню та інтерактивні стани Pullora використовують один набір правил замість локальних CSS-виправлень. Візуальний шар дозволено повністю перезбирати, але чинні сервіси, hooks, Tauri-команди, download queue, rollback і SHA-256 перевірки залишаються джерелом поведінки.

> Історичний запис. Реалізація спричинила неприйнятний візуальний дрейф і була повністю відкочена у `v5.10.1`. Позначки `[x]` нижче означають лише те, що робота входила до `v5.10.0`, а не те, що її реалізація збереглася після відкату. Її підхід із одночасною заміною всього візуального шару не можна повторювати; чинним джерелом вигляду є дизайн-контракт вище.
