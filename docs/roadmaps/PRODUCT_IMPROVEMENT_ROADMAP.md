# Roadmap покращення Pullora

Оновлено: 17 липня 2026 року.

Цей файл є джерелом правди для запланованих покращень Pullora після `v5.2.43`. План не залежить від історії чатів або пам'яті асистента.

## Основні правила

- Українська є мовою за замовчуванням.
- Усі нові тексти повинні мати український та англійський переклади.
- Не змінювати усталений темний дизайн без окремого рішення.
- Спочатку надійність і безпека, потім нові можливості.
- Не додавати залежності, якщо завдання вирішується засобами React, TypeScript, Rust або Tauri.
- Кожен користувацький реліз повинен пройти production-збірку, smoke-test і перевірку обох мов.

## v5.2.44 — стабілізація

Мета: випустити поточні виправлення без зміни візуального напрямку Pullora.

- [x] Обмежити висоту карток деталей і коректно обрізати довгий шлях встановлення.
- [x] Увімкнути Content Security Policy для Tauri.
- [x] Обмежити локальний asset protocol папками з обкладинками Pullora.
- [x] Дозволити завантаження лише з GitHub Releases відповідного репозиторію.
- [x] Дозволити самооновлення лише з офіційного репозиторію Pullora.
- [x] Видалити невикористаний shell-плагін і його дозвіл.
- [x] Оновити Vite до безпечної гілки 7.
- [x] Замінити дубльовані `useSettings()` на один `SettingsProvider`.
- [x] Додати Rust-тести перевірки release URL.
- [x] Перевірити зібраний frontend через headless Playwright на розмірах 1000×700, 1280×720 і 1920×1080.
- [x] Перевірити українську й англійську мови та рівність 772 ключів перекладу.
- [x] Підготувати production-артефакти й успішно пройти release-check та smoke-test portable EXE.
- [x] Створити commit, тег і GitHub Release.

Готово, коли:

- картки не розтягуються через довгі значення;
- production-збірка Tauri запускається;
- `npm audit --omit=dev` не має відомих вразливостей;
- усі Rust-тести проходять;
- обидві мови перевірені в застосунку.

## v5.3.0 — локалізація і помилки

Мета: жодних пошкоджених рядків, сирих HTTP-помилок або текстів лише однією мовою.

- [x] Замінити публічні текстові помилки Rust стабільними кодами; внутрішні технічні деталі залишати лише в логах.
- [x] Перекладати коди помилок у React через чинний i18n-шар.
- [x] Перевірити UTF-8 українських рядків у Rust і документації; пошкоджених файлів не виявлено.
- [x] Перевірити кнопки, підказки, діалоги, порожні стани та повідомлення про помилки статичним аудитом і headless smoke-test для `uk` та `en`.
- [x] Додати автоматичний тест рівності ключів `uk` та `en`.
- [x] Прибрати дублікати ключів у словниках.
- [x] Локалізувати NSIS-інсталятор українською та англійською.
- [x] Форматувати дати, числа й розміри відповідно до вибраної мови.

Готово, коли:

- перемикання мови не залишає текстів іншою мовою;
- UI не показує внутрішні Rust, Tauri або HTTP-повідомлення;
- CI відхиляє неповні переклади.

## v5.4.0 — безпечне самооновлення

Мета: Pullora встановлює лише перевірені оновлення й не зберігає секрети відкритим текстом.

- [x] Перенести GitHub token із `config.json` у Windows Credential Manager.
- [x] Передбачити системне сховище секретів для інших платформ.
- [x] Додати SHA-256 для release-артефактів Pullora.
- [x] Перевіряти джерело, SHA-256 і тип файла до запуску.
- [x] Обмежити файлові команди каталогами Pullora та вибраною папкою встановлення.
- [x] Заборонити самооновлення після будь-якої невдалої перевірки.
- [x] Зберігати останній робочий бінарник для відновлення.

Готово, коли змінений файл неможливо встановити як оновлення Pullora.

## v5.5.0 — надійне встановлення

Мета: перерване або невдале встановлення не пошкоджує активну версію застосунку.

- [x] Формалізувати стани: завантаження, перевірка, розпакування, встановлення, перевірка запуску, завершення.
- [x] Автоматично очищати неповні інсталяції.
- [x] Відновлювати попередню версію після помилки.
- [x] Додати повтор невдалого завантаження.
- [x] Обмежити кількість паралельних завантажень.
- [x] Показувати розмір, архітектуру та тип release asset.
- [x] Дозволити пропустити конкретну версію оновлення.
- [x] Показувати release notes до підтвердження оновлення.

Готово, коли примусове переривання кожного етапу залишає реєстр і активну версію у валідному стані.

## v5.6.0 — декомпозиція бібліотеки

Мета: зміна однієї панелі більше не повинна ламати геометрію сусідніх блоків.

Поступово винести з `LibraryPage.tsx`:

- [x] `LibrarySidebar`;
- [x] `LibraryHero`;
- [x] `VersionPanel`;
- [x] `ApplicationDetails`;
- [x] `FolderManager`;
- [x] `BatchUpdatePanel`;
- [x] hooks вибору, фільтрації та оновлень:
  - [x] вибір активного застосунку;
  - [x] фільтрація й сортування;
  - [x] масові оновлення.

Додатково:

- [x] Централізувати повторювані обчислення статусів.
- [x] Додати компонентні тести основних панелей.
- [x] Зафіксувати мінімальні й максимальні розміри карток.
- [x] Не вводити нову глобальну state-бібліотеку без доведеної потреби.

Архітектурне рішення: локальний стан компонентів і тематичні hooks покривають поточні сценарії; нова глобальна state-бібліотека не потрібна.

Готово, коли компоненти можна тестувати окремо, а `LibraryPage` відповідає переважно за компонування сторінки.

## v5.7.0 — UX бібліотеки

Мета: основні дії доступні не більше ніж за два кліки.

- [x] Режими «Усі», «Встановлені», «Оновлення» та «Обране».
- [x] Компактний і звичайний вигляд.
- [x] Сортування за назвою, запуском, встановленням та оновленням.
- [x] Розмір застосунку на диску.
- [x] Контекстне меню: запуск, папка, оновлення, версії, видалення.
- [x] Запам'ятовування вибраного застосунку, фільтрів і позиції прокручування.
- [x] Підтвердження масового оновлення.
- [x] Віртуалізація або `content-visibility` лише якщо вимірювання покаже проблему на великих бібліотеках.
  - Headless-бенчмарк 1000 карток: 129,8 мс без оптимізації та 47,95 мс із `content-visibility`; оптимізацію застосовано лише до карток сайдбара.

Мета: застосунком можна користуватися без миші, а діагностичний звіт не містить секретів.

- [x] Повна навігація клавіатурою.
- [x] Помітний `focus-visible`.
- [x] Focus trap і повернення фокуса після закриття діалогів.
- [x] `aria-live` для прогресу, помилок і завершення операцій.
- [x] Підтримка `prefers-reduced-motion`.
- [x] Автоматична перевірка контрасту тем.
- [x] Екран журналу подій.
- [x] Кнопка «Скопіювати діагностику».
- [x] Автоматичне приховування token і персональних шляхів.
- [x] Ротація старих логів.

## v5.9.0 — масові дії бібліотеки

Мета: виконувати однакову дію для кількох застосунків без повторення операції для кожної картки.

- [x] Додати Ctrl/Shift-вибір, Ctrl+A, Ctrl/Shift+Space та Escape.
- [x] Показувати кількість вибраних і панель дій унизу сайдбара.
- [x] Оновлювати, переміщувати до папки та змінювати стан «Обране» для вибраних застосунків.
- [x] Видаляти вибрані встановлені застосунки після одного підтвердження.
- [x] Очищати всі неактивні версії, не видаляючи активну.
- [x] Показувати локалізований підсумок успішних, пропущених і невдалих операцій.
- [x] Залишати невдалі елементи вибраними для повтору.
- [x] Перевірити клавіатуру, обидві мови, теми, щільності та підтримувані розміри вікна.

Готово, коли всі масові дії використовують чинні безпечні Tauri-команди, панель не перекриває список, а часткова помилка не приховує невдалі елементи.
