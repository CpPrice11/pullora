# Pullora v5.2.1

## Українська

- Прибрано автоматичні GitHub-запити під час запуску Store і Library.
- Додано постійний кеш відповідей та ETag між перезапусками Pullora.
- Додано `304 Not Modified`, stale fallback і захист від вичерпання квоти.
- GitHub-запити проходять через пріоритетну чергу з максимумом двох одночасних операцій.
- Застарілі пошуки видаляються з черги, а результати вже активних старих пошуків ігноруються.
- У Settings → Maintenance показуються квоти Core/Search API та стан черги.

## English

- Removed automatic GitHub requests during Store and Library startup.
- Added persistent response and ETag caching across Pullora restarts.
- Added `304 Not Modified`, stale fallback, and rate-limit protection.
- GitHub requests now use a priority queue with at most two concurrent operations.
- Obsolete searches are removed from the queue and stale active search results are ignored.
- Settings → Maintenance now shows Core/Search API quotas and queue state.
