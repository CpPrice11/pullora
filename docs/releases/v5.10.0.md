# Pullora v5.10.0

## Українська

- Запроваджено єдину дизайн-систему із семантичними токенами, спільними поверхнями, кнопками, полями, статусами, меню, списками, діалогами та порожніми станами.
- Темна й світла теми тепер використовують однакову структуру компонентів; додано автоматичну перевірку, яка не дозволяє темі змінювати геометрію інтерфейсу.
- Повністю уніфіковано контекстні та overflow-меню: вони відкриваються через portal, не обрізаються панелями, враховують краї вікна та підтримують клавіатуру.
- Сортування, тема, мова та стратегія вибору файла переведені на доступний `UiSelect` із typeahead і поверненням фокуса.
- Розділено загальний фон кожної теми, hero-фон і обкладинку застосунку: тепер вони не замінюють одне одного.
- Перевірено світлу палітру, кнопку «Запустити», disabled/destructive-стани, читабельність поверхонь і WCAG-контраст.
- Видалено старі оболонки `.sam-shell` і `.cinematic-shell`, а CSS розкладено на канонічні шари.

Цей реліз не використовує Authenticode Code Signing. Цілісність файлів перевіряється через `SHA256SUMS.txt`.

## English

- Introduced a unified design system with semantic tokens and shared surfaces, buttons, fields, statuses, menus, selects, dialogs, and empty states.
- Dark and light themes now share the same component structure; an automated check prevents theme rules from changing interface geometry.
- Unified context and overflow menus through a portal-based overlay that avoids clipping, handles viewport edges, and supports keyboard navigation.
- Moved library sorting, theme, language, and asset strategy to the accessible `UiSelect` with typeahead and focus restoration.
- Separated each theme's global background, application hero background, and cover so they no longer replace one another.
- Audited the light palette, Launch button, disabled/destructive states, surface readability, and WCAG contrast.
- Removed the legacy `.sam-shell` and `.cinematic-shell` shells and organized CSS into canonical layers.

This release does not use Authenticode Code Signing. File integrity is verified through `SHA256SUMS.txt`.
